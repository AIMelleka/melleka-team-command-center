## Paid Advertising (Google Ads, Meta, LinkedIn, TikTok)

### Platform Selection
| Platform | Best For | Use When |
|----------|----------|----------|
| Google Ads | High-intent search traffic | People actively search for your solution |
| Meta | Demand generation, visual products | Creating demand, strong creative assets |
| LinkedIn | B2B, decision-makers | Job title/company targeting, higher price points |
| TikTok | Brand awareness, younger demographics | Audience skews 18-34, video capacity |

### Campaign Structure
- Name campaigns: `[Platform]_[Objective]_[Audience]_[Offer]_[Date]`
- Testing phase (weeks 1-4): 70% proven campaigns, 30% testing
- Scaling: increase budgets 20-30% at a time, wait 3-5 days between increases
- Always exclude existing customers and recent converters

### Ad Copy Frameworks
- **PAS (Problem-Agitate-Solve)**: Problem → Agitate pain → Solution → CTA
- **BAB (Before-After-Bridge)**: Current painful state → Desired future → Your product as bridge
- **Social Proof Lead**: Impressive stat/testimonial → What you do → CTA

### Optimization Levers
- High CPA → Check landing page first, tighten targeting, test new creative angles
- Low CTR → Creative isn't resonating, test new hooks, check audience fit
- High CPM → Audience too narrow, try different placements
- Bid strategy: Start manual → Gather 50+ conversions → Switch to automated

### Retargeting
| Stage | Window | Message |
|-------|--------|---------|
| Hot (cart/trial) | 1-7 days | Urgency, objection handling |
| Warm (key pages) | 7-30 days | Case studies, demos |
| Cold (any visit) | 30-90 days | Educational, social proof |

## Ad Creative Generation

### Platform Character Limits (Always Enforce)
| Platform | Headline | Description |
|----------|----------|-------------|
| Google RSA | 30 chars (up to 15) | 90 chars (up to 4) |
| Meta | 40 chars recommended | 30 chars recommended |
| LinkedIn | 70 chars recommended | 100 chars recommended |
| TikTok | 80 chars recommended | - |

### Creative Angle Categories
| Category | Example |
|----------|---------|
| Pain point | "Stop wasting time on X" |
| Outcome | "Achieve Y in Z days" |
| Social proof | "Join 10,000+ teams who..." |
| Curiosity | "The X secret top companies use" |
| Comparison | "Unlike X, we do Y" |
| Urgency | "Limited time: get X free" |
| Identity | "Built for [specific role/type]" |

### Writing Quality
- Specific ("Cut reporting time 75%") over vague ("Save time")
- Benefits ("Ship code faster") over features ("CI/CD pipeline")
- Numbers when possible ("3x faster", "in 5 minutes", "10,000+ teams")
- Always validate creative against platform character limits before delivering

### Iteration Loop
1. Pull performance data → 2. Identify winning patterns (themes, structures, words) → 3. Generate new variations doubling down on winners → 4. Test 1-2 new angles → 5. Retire underperformers

## Copywriting

### Core Principles
1. **Clarity over cleverness** — If you must choose, choose clear
2. **Benefits over features** — What it means for the customer, not what it does
3. **Specificity over vagueness** — "Cut weekly reporting from 4 hours to 15 minutes" not "Save time"
4. **Customer language** — Mirror voice-of-customer from reviews, interviews, support tickets
5. **One idea per section** — Build a logical flow down the page

### Writing Style Rules
- Simple over complex: "Use" not "utilize", "help" not "facilitate"
- Active over passive: "We generate reports" not "Reports are generated"
- Confident: Remove "almost", "very", "really"
- No exclamation points, no marketing buzzwords without substance

### Headline Formulas
- "{Achieve outcome} without {pain point}"
- "The {category} for {audience}"
- "Never {unpleasant event} again"
- "{Question highlighting main pain point}"

### CTA Copy
- Weak: Submit, Sign Up, Learn More, Click Here, Get Started
- Strong: Start Free Trial, Get [Specific Thing], See [Product] in Action
- Formula: [Action Verb] + [What They Get] + [Qualifier]

### Page Structure (Above the Fold)
1. Headline (core value proposition, specific)
2. Subheadline (expands on headline, 1-2 sentences)
3. Primary CTA (action-oriented, communicates what they get)

### Core Page Sections
| Section | Purpose |
|---------|---------|
| Social Proof | Logos, stats, testimonials |
| Problem/Pain | Show you understand their situation |
| Solution/Benefits | 3-5 key benefits connected to outcomes |
| How It Works | 3-4 steps to reduce complexity |
| Objection Handling | FAQ, comparisons, guarantees |
| Final CTA | Recap value, repeat CTA, risk reversal |

## SEO Audit & Optimization

### Audit Priority Order
1. Crawlability & Indexation (can Google find it?)
2. Technical Foundations (fast and functional?)
3. On-Page Optimization (content optimized?)
4. Content Quality (deserves to rank?)
5. Authority & Links (has credibility?)

### Technical SEO Checklist
- Robots.txt: No unintentional blocks, sitemap referenced
- XML Sitemap: Accessible, submitted, only canonical indexable URLs
- Important pages within 3 clicks of homepage
- Core Web Vitals: LCP < 2.5s, INP < 200ms, CLS < 0.1
- HTTPS everywhere, valid SSL, no mixed content
- Responsive design, proper viewport, no horizontal scroll

### On-Page SEO
- Title tags: Unique, primary keyword near beginning, 50-60 chars, brand at end
- Meta descriptions: Unique, 150-160 chars, keyword + value prop + CTA
- One H1 per page with primary keyword, logical H2→H3 hierarchy
- Keyword in first 100 words, related keywords naturally used
- Alt text on all images, descriptive file names
- Internal linking: Important pages well-linked, descriptive anchor text

### Content Quality (E-E-A-T)
- Experience: First-hand, original insights/data
- Expertise: Author credentials visible, accurate detail
- Authoritativeness: Recognized, cited by others
- Trustworthiness: Transparent, contact info, secure site

### Common Issues by Site Type
- **SaaS**: Thin product pages, blog not connected to product, missing comparison pages
- **E-commerce**: Thin categories, duplicate descriptions, missing product schema
- **Content/Blog**: Outdated content, keyword cannibalization, poor internal linking
- **Local Business**: Inconsistent NAP, missing local schema, no Google Business Profile

## AI SEO (AEO/GEO/LLMO)

### How AI Search Works
- AI Overviews appear in ~45% of Google searches, reducing clicks by up to 58%
- Traditional SEO gets you ranked; AI SEO gets you **cited**
- Well-structured pages can get cited even from page 2-3

### Three Pillars
1. **Structure (Extractable)**: Lead sections with direct answers, 40-60 word answer passages, tables beat prose for comparisons, numbered lists for processes
2. **Authority (Citable)**: Cite sources (+40% visibility), add statistics (+37%), expert quotes (+30%), authoritative tone (+25%)
3. **Presence (Findable)**: Wikipedia mentions, Reddit discussions, industry publications, review sites, YouTube

### Content That Gets Cited Most
| Type | Citation Share |
|------|:------------:|
| Comparison articles | ~33% |
| Definitive guides | ~15% |
| Original research/data | ~12% |
| Best-of/listicles | ~10% |
| How-to guides | ~8% |

### Schema Markup for AI
- Articles/Blog: `Article`, `BlogPosting`
- How-to: `HowTo`
- FAQs: `FAQPage`
- Products: `Product`
- Comparisons: `ItemList`

## Content Strategy

### Searchable vs Shareable
- **Searchable**: Captures existing demand. Target keyword, match intent, comprehensive.
- **Shareable**: Creates demand. Novel insight, original data, challenges conventional wisdom.
- Prioritize searchable first — it's the foundation.

### Content Types
- **Use-Case Content**: [persona] + [use-case] (long-tail keywords)
- **Hub and Spoke**: Hub = comprehensive overview, Spokes = subtopics, interlinked
- **Template Libraries**: High-intent keywords + product adoption
- **Thought Leadership**: Articulate concepts everyone feels but hasn't named
- **Data-Driven**: Product data analysis, original research
- **Case Studies**: Challenge → Solution → Results → Key learnings

### Keyword Research by Buyer Stage
| Stage | Modifiers | Content Focus |
|-------|-----------|---------------|
| Awareness | "what is", "how to", "guide to" | Educational |
| Consideration | "best", "vs", "alternatives", "comparison" | Evaluation |
| Decision | "pricing", "reviews", "demo", "trial" | Conversion |
| Implementation | "templates", "tutorial", "how to use" | Activation |

### Prioritizing Content
1. Customer Impact (40%): Frequency, emotional charge, LTV potential
2. Content-Market Fit (30%): Alignment with product, unique insights
3. Search Potential (20%): Volume, competition, growth trend
4. Resource Requirements (10%): Expertise, research needed

## Page CRO (Conversion Rate Optimization)

### Analysis Framework (Impact Order)
1. **Value Proposition Clarity**: Can visitor understand what & why in 5 seconds?
2. **Headline Effectiveness**: Communicates core value? Specific enough? Matches traffic source?
3. **CTA**: One clear primary action? Visible above fold? Button copy communicates value?
4. **Visual Hierarchy**: Scannable? Important elements prominent? Enough white space?
5. **Trust Signals**: Logos, testimonials, case studies, review scores near CTAs
6. **Objection Handling**: FAQ, guarantees, comparisons, process transparency
7. **Friction Points**: Too many form fields, unclear next steps, mobile issues, slow load

### Page-Specific CRO
- **Homepage**: Clear positioning, quick path to conversion, handle "ready" AND "researching"
- **Landing Page**: Single CTA, message match with traffic source, complete argument on page
- **Pricing Page**: Clear comparison, recommended plan obvious, address "which plan?" anxiety
- **Feature Page**: Connect feature → benefit → outcome, show use cases

## Marketing Psychology (70+ Mental Models)

### Pricing Psychology
- **Anchoring**: Show higher price first to set expectations
- **Decoy Effect**: Add inferior 3rd option to make preferred option obvious
- **Charm Pricing**: $99 feels much cheaper than $100 (left-digit effect)
- **Rule of 100**: Under $100 use %, over $100 use $ amounts for discounts
- **Mental Accounting**: "$3/day" feels cheaper than "$90/month"
- **Good-Better-Best**: 3 tiers where middle is target

### Persuasion Principles
- **Reciprocity**: Give value first (free content, tools, trials)
- **Social Proof**: Customer counts, testimonials, logos, reviews
- **Scarcity/Urgency**: Limited availability (only when genuine)
- **Loss Aversion**: Frame what they'll lose by not acting
- **Commitment & Consistency**: Get small commitments first (email → trial → paid)
- **Authority**: Expert endorsements, certifications, "featured in" logos

### Behavior Models
- **BJ Fogg**: Behavior = Motivation x Ability x Prompt (need all three)
- **Endowment Effect**: Free trials make people feel ownership
- **Goal-Gradient**: Progress bars drive completion
- **Paradox of Choice**: Fewer options = more decisions (3 tiers beat 7)
- **Status-Quo Bias**: Reduce friction to switch ("Import data in one click")

### Quick Reference
| Challenge | Apply |
|-----------|-------|
| Low conversions | Reduce friction, simplify choices, clear CTA |
| Price objections | Anchoring, framing, mental accounting |
| Building trust | Social proof, authority, reciprocity |
| Increasing urgency | Scarcity, loss aversion |
| Retention/churn | Endowment effect, switching costs |
| Decision paralysis | Fewer options, defaults, nudges |

## Email Marketing & Sequences

### Sequence Types & Length
| Type | Length | Goal |
|------|--------|------|
| Welcome | 5-7 emails, 12-14 days | Activate, build trust, convert |
| Lead Nurture | 6-8 emails, 2-3 weeks | Demonstrate expertise, convert |
| Re-Engagement | 3-4 emails, 2 weeks | Win back or clean list |
| Onboarding | 5-7 emails, 14 days | Activate, drive to aha moment |

### Email Copy Rules
- One email, one job, one CTA
- Value before ask — lead with usefulness
- Structure: Hook → Context → Value → CTA → Sign-off
- Short paragraphs (1-3 sentences), mobile-first
- Subject lines: Clear > Clever, 40-60 chars, benefit or curiosity-driven

### Subject Line Patterns
- Question: "Still struggling with X?"
- How-to: "How to [outcome] in [timeframe]"
- Number: "3 ways to [benefit]"
- Direct: "[Name], your [thing] is ready"
- Story: "The mistake I made with [topic]"

## A/B Testing

### Sample Size Rules
- Minimum 1,000 visitors per variation for meaningful results
- Run tests for full business cycles (minimum 2 weeks)
- Don't peek — wait for statistical significance (95%)
- Test one variable at a time for clear learnings

### Testing Hierarchy (Highest Impact First)
1. Concept/angle (what you say)
2. Hook/headline
3. Visual style
4. Body copy
5. CTA

## Churn Prevention

### Dunning (Failed Payments)
- Send 3-4 emails over 7-14 days before cancellation
- Include direct "update payment" link
- Escalate urgency: helpful → reminder → warning → last chance

### Cancel Flow
- Ask why (required, select from options)
- Offer relevant save: discount, pause, downgrade, support
- Confirm cancellation gracefully — don't make it hard

## When a Team Member Asks for Marketing Help

**Always be proactive with tools:**
- Need to know ad performance? → Pull data with google_ads_query or supermetrics_query first
- Need to check SEO? → Use semrush_query for domain overview, keywords, backlinks
- Need to write content? → Research the topic first, check competitors, pull relevant data
- Need to send communications? → Draft, show for approval, then send via send_email or slack_post
- Need to create a landing page? → Build it in scratch dir, deploy with deploy_site
- Need to set up automation? → Create cron jobs for recurring tasks
- Need analytics data? → Use supermetrics_query with the right data source ID
- Need to find leads or contacts? → Use apollo_search to find people by title, company, location, industry
- Need to enrich a contact or company? → Use apollo_enrich with an email, name+domain, or company domain

**Always think in terms of:**
1. What data do I need? (Pull it with tools first)
2. What's the strategy? (Apply marketing frameworks)
3. What's the execution? (Use tools to implement)
4. What should we measure? (Set up tracking/cron jobs for monitoring)
